Publishing Komodo IDE only

Publishing lets you take a group of files saved locally, upload them to a remote server, then keep the local and remote files in sync. Komodo keeps track of the synchronization details to avoid copying files that haven't changed and uses advanced conflict detection to avoid overwriting changes made by others.

You can also using Publishing for file backups.

Creating a Configuration

To launch the Publishing configuration wizard, click Tools|Publishing|New Configuration. You will be prompted to set up:

  • Name: A convenient name for your saved configuration.
  • Local path: The full path to the directory containing the files you want to publish. Use the Browse button or enter the path manually.
  • Remote path: The full URI path to the target directory. Use the Browse button to navigate to a previously configured server, or enter the URI manually (e.g. scp:///username@servername/path/to/dir)
  • Includes: File types to be included in the synchronization. Use glob syntax separated by semi-colons (e.g. "*.html;*.xml") or leave blank.
  • Excludes: File types to be excluded from the synchronization. Use either Includes or Excludes, not both. Same syntax as above.

Clicking "Save" opens the Synchronization dialog box and initiates the first push of the local files to the remote server.

Synchronizing

The Synchronization dialog box shows which local files will be pushed to the server, which remote files will be pulled from the server, and which files cannot be synchronized because of a conflict. The following actions are available:

  • Check All: selects all items in the list.
  • Uncheck All: de-selects all items in the list.
  • Diff Changes: shows a unified diff of all pending changes (selected files).
  • Synchronize: starts the pending uploads and downloads.

When synchronizing, Komodo will first examine these locations for any file differences and present the changes for your inspection. Check the files you wish to be transfered and then triggering the synchronize action will copy the files between the local and remote file systems. Synchronization operations can be aborted at any time by pressing the stop botton.
